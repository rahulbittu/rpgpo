# TopRanker Constitution Bridge

## Purpose

TopRanker is a child office and flagship mission inside RPGPO.

TopRanker may maintain its own constitution, principles, coding standards, product values, and operating rules.
Those should be respected and preserved.

RPGPO should not overwrite TopRanker's own constitution.
Instead, RPGPO should integrate it through inheritance and bridge rules.

---

## Inheritance Model

### Parent Layer
RPGPO governs:
- privacy
- approvals
- logging
- access control
- human sovereignty
- memory architecture
- risk classification
- board orchestration

### Child Layer
TopRanker governs:
- product philosophy
- product-specific values
- user value interpretation
- implementation style preferences
- product positioning
- internal product quality rules
- domain-specific workflows

---

## Conflict Rule

If a TopRanker rule conflicts with:
- privacy policy
- approval policy
- access policy
- human sovereignty
- system-level risk rules

Then RPGPO rules win.

If the question is product-specific and does not violate RPGPO governance, then TopRanker rules win inside the TopRanker domain.

---

## Integration Rule

RPGPO should always try to ingest and honor:
- TopRanker constitution docs
- TopRanker README
- TopRanker architecture notes
- TopRanker roadmap docs
- TopRanker coding conventions
- TopRanker product strategy docs

These should be treated as domain memory for the TopRanker child office.

---

## Operating Principle

TopRanker is not just another folder.
It is the flagship business mission.

RPGPO should use TopRanker's own intelligence and documentation as a source of domain truth while still maintaining global governance above it.
