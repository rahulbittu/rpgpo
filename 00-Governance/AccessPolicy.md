# RPGPO Access Policy

## Default Rule
Unconnected means unavailable.

## Access Classes

### Class A: Core Safe Access
- governance docs
- approved local folders
- approved repos
- dashboard data
- public sources

### Class B: Controlled Operational Access
- dedicated Gmail inboxes
- selected socials
- project management tools
- approved browser sessions

### Class C: Sensitive Scoped Access
- single exported email threads
- selected private documents
- selected financial research materials
- narrow personal context explicitly shared

### Class D: Forbidden or Unconnected Access
Everything not explicitly approved.

## Operational Rules
- Broad mailbox access is disallowed by default.
- Full personal account access is disallowed by default.
- Access should be granted per workflow, not by vague trust.
- High-sensitivity data should be shared through exports, not blanket account access.

## Topranker Gmail Rule
`topranker@gmail.com` is the controlled system inbox.
Selected context can be forwarded or exported into it.
The rest of the personal inbox remains outside scope.
