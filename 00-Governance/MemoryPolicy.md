# RPGPO Memory Policy

## Principle
Memory is useful only when it is scoped, inspectable, and intentionally granted.

## Layers
1. Constitutional Memory
2. Rahul Preference Memory
3. Domain Memory
4. Session / Working Memory
5. Sensitive Scoped Memory

## Constitutional Memory
Includes:
- identity
- values
- governance
- board roles
- approval policy
- access classes
- logging rules

## Rahul Preference Memory
Includes only long-lived preferences and standards likely to matter repeatedly.
Examples:
- preferred communication style
- favored product standards
- recurring priorities
- tone preferences
- strategic goals

## Domain Memory
Stored separately per office/domain:
- TopRanker
- Founder strategy
- Career
- Wealth research
- Creative work

## Session Memory
Temporary work context for current tasks.
Should be disposable and not automatically promoted to long-term memory.

## Sensitive Scoped Memory
Examples:
- exact Gmail thread exports
- selected private documents
- selected financial notes
- personal communications approved for narrow use

## Promotion Rules
Information should move upward into longer-lived memory only when:
- it is repeatedly useful
- it is slow-changing
- it is explicitly approved or clearly operationally necessary

## Isolation Rules
- Sensitive scoped memory must not leak into unrelated tasks.
- Domain memories must not be merged casually.
- Child offices inherit constitutional memory, not unrestricted domain memory.

## Deletion Principle
If memory is no longer useful or should no longer be retained, remove it rather than preserve clutter.
