## GTD-Style Project Reviews: Best Practices and Methodologies

**Core GTD weekly review involves auditing projects, next actions, contexts, and inactive items to maintain system integrity, typically done in 1-2 hours every weekend.** Best practices emphasize built-in app tools for surfacing stalled projects (no next actions), overdue tasks, and automated checklists; manual audits fail for 30+ projects.[1][2]

- **Weekly Review Checklist from GTD Experts**: Get current with inbox (process all items), review calendars (past week + next 12 months), scan all projects/projects lists (ensure 1+ next action per active project), check contexts/next actions (mark stalled items), and review someday/maybe (advance or delete). David Allen stresses flexibility—if skipped, do mini-reviews mid-week; frustration comes from rigidity, not the practice itself.[2]
- **App-Enabled Reviews**: OmniFocus tracks last review date per project, flags overdue/stale items, and offers forecast views merging tasks/calendars; use perspectives for custom filters like "@errands <15min". Other tools like Things 3 support areas/projects for GTD separation.[1][7]
- **Real Example**: Obsidian GTD Dashboard aggregates active projects via Dataview queries (e.g., `#project` + `status: active`, sorted by priority/area); inbox shows unprocessed daily notes (e.g., 3 backlog notes trigger catch-up). Used in AI-human hybrid systems for visual project oversight.[3]

## Energy-Based Scheduling: Best Practices and Integration with GTD

**No direct 2026 sources in results link energy-based scheduling (e.g., matching high-energy tasks to peak times) explicitly to GTD; GTD traditionally avoids fixed time-blocking to stay flexible, but hybrid approaches use defer dates/contexts for energy alignment.** Conflicts noted: Pure GTD debates time-blocking as "breaking rules" vs. boosting output; apps like OmniFocus enable it via flexible deferral.[1][2]

- **GTD-Compatible Energy Practices**: Tag tasks by energy/context (e.g., @high-energy for deep work, defer low-energy to evenings); forecast views in OmniFocus stack calendar + tasks for energy-aware planning without rigid blocks. Avoid flat lists—use project hierarchies to prioritize by energy fit.[1]
- **Hybrid Methodology**: Separate active (high-energy now) vs. inactive projects; weekly reviews cull low-energy "someday/maybe". David Allen applied this to renovate an apartment: tracked next actions by energy/context, not calendar slots.[2]
- **Real Example**: In Things 3/Blitzit comparisons, GTD users choose tools with areas/projects + focus modes; Blitzit adds time-tracking for energy audits (e.g., log deep work sessions), cross-platform for non-Apple energy scheduling.[7]
